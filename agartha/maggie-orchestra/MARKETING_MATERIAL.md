# Symphony OS Marketing Material

## Positioning statement

Symphony OS is a human-first multi-agent operating layer that turns AI from a chat tool into coordinated, role-specific digital labor with persistent cross-device presence.

## One-line value prop

Run your own AI team like a company org chart, with faster retrieval, safer execution, and resident companions users can actually work with.

## Audience segments

1. Public sector and education
- Accessibility-first communication
- Observer-safe XR demonstrations
- Workforce and civic-tech pilots

2. Mid-market and enterprise operations
- Multi-role orchestration across go-to-market and product teams
- Retrieval efficiency and reduced model waste
- Clear auditability and handoff structure

3. Creator and immersive ecosystem
- Browser/desktop resident companions
- Spatial and narrative co-presence
- Personalized persona networks

## Core differentiators

- Agent-specific roles with explicit mission boundaries
- Mercury-first messenger lane for low-latency retrieval triage
- Pre-ingest document chunking before agent consumption
- Placement-aware resident system for real interaction contexts
- Scenario simulation outputs for decision storytelling

## Messaging pillars

1. Sovereignty
- Keep orchestration and control in your own stack

2. Accessibility
- Build communication tools that increase voice and agency

3. Operational trust
- Controlled friction where it matters: permissions, policy, spend, and safety

4. Measurable value
- Track delta in speed, quality, and cost from baseline workflows

## Objection handling

“Is this just another chatbot?”
- No. It is a multi-agent operating layer with routing, retrieval, resident interfaces, and stateful orchestration.

“Will this replace teams?”
- It augments teams by automating repetitive coordination and accelerating specialist workflows.

“Is it safe?”
- It supports controlled friction at trust boundaries and can run with local governance constraints.

## Pitch track (7-minute)

1. Problem (60s)
- Teams are fragmented across tools and model calls; context is wasted and execution is brittle.

2. Solution (90s)
- Symphony assigns role-specific agents and routes tasks through explicit lanes.

3. Live flow (120s)
- Show resident placement + agent switch + retrieval route + doc preprocess.

4. Impact story (90s)
- Accessibility and communication support with observer context.

5. Business model (60s)
- Subscription platform + implementation + vertical modules.

6. Pilot ask (60s)
- 30-day scoped deployment with KPI review and expansion path.

## Pricing model template

- Starter: core orchestration + 5 agents + API controls
- Pro: advanced routing + resident system + simulation outputs
- Enterprise: SSO, governance controls, custom integrations, dedicated support

## Pilot offer template

- Duration: 30 days
- Scope:
  - role mapping for one department
  - retrieval pipeline enabled
  - resident interface configured
  - KPI dashboard and final readout
- KPI examples:
  - response and handoff latency
  - doc retrieval efficiency
  - task completion rate
  - user trust/satisfaction
